Neural Machine Translation · 75+ Languages · Pay-per-character · Fully Managed

You send text in, you get translated text back. It's as simple as that — which is exactly why the exam loves it.

How It Works — Real-Time vs. Batch

🛒 Real-World Use Case
Global E-Commerce — Product Catalogue in 20 Languages

An e-commerce company has 500,000 product descriptions in English. They need to launch in 20 countries within 3 months. Instead of hiring 20 translation agencies, they trigger an S3 event notification when a product description is saved — a Lambda function calls Amazon Translate with a custom terminology file (so "Prime Membership" is never mistranslated) and writes translated versions back to S3. Total infrastructure: 1 Lambda function, 1 Translate API call, $0.015 per 1000 characters. No ML team hired.

🚨 Exam Trap — Translate ≠ Transcribe: Amazon Translate converts text between languages. Amazon Transcribe converts speech to text. The exam will describe "call centre audio in Spanish → convert to English text" and try to trick you. That's Transcribe first (speech→text) then Translate (Spanish→English). Two services. One step each.

Amazon Elastic Transcoder
Managed Media Conversion · S3-to-S3 · Preset-based · Pay-per-minute of video

Elastic Transcoder is a fully managed media transcoding service. It converts video and audio files stored in S3 into formats optimised for playback on phones, tablets, desktops, and smart TVs. Think: you upload a 4K .MOV from a camera and it outputs a 1080p MP4, a 720p MP4, and a 360p MP4 automatically using presets. Zero servers to manage.

💡 Key Concept — Pipelines, Jobs & Presets:
Pipeline = The queue that manages your transcoding jobs (input S3 bucket, output S3 bucket, IAM role).
Job = One transcoding task — "take this file, apply these presets, output here."
Preset = A template defining codec, resolution, bitrate, container format. AWS provides presets for iPhone, Android, Web, Fire TV, etc.

📺 Real-World Use Case
Online Learning Platform — Lecture Video Processing

An online education startup allows instructors to upload lecture videos. Raw files arrive in S3 as large .MP4 or .MOV files. An S3 event triggers a Lambda function that creates an Elastic Transcoder job with three output presets — 1080p for desktop, 720p for tablet, 360p for mobile. Students always get the right quality for their device. Cost: ~$0.015/minute of transcoded video.

AWS CloudHSM
Hardware Security Module · FIPS 140-2 Level 3 · You Own the Keys · Single-Tenant

CloudHSM gives you a dedicated hardware security module (HSM) device inside AWS that only you control. Unlike KMS (where AWS manages the hardware and can technically access keys), CloudHSM means AWS has zero access to your keys. The hardware is yours. The key material never leaves the HSM. Required for the strictest regulatory requirements.

💡 KMS vs CloudHSM — The Mental Model:
KMS = AWS-managed key service. AWS operates the hardware. Multi-tenant. AWS can comply with law enforcement requests.
CloudHSM = You rent dedicated hardware. AWS cannot see or access your keys. Single-tenant HSM. FIPS 140-2 Level 3 certified.

Level 3 Certified

~$1.60/hr

Architecture — CloudHSM Cluster for Database Encryption

🏦 Real-World Use Case
Investment Bank — Regulatory Requirement for HSM

A global investment bank is subject to regulations requiring that encryption keys for trading system data are stored in FIPS 140-2 Level 3 certified hardware that the bank exclusively controls — not shared with any cloud provider. They deploy a 2-node CloudHSM cluster across two AZs. Their Oracle databases use Transparent Data Encryption (TDE) with keys stored exclusively in CloudHSM. AWS cannot access the keys under any circumstance — not even with a subpoena, because the keys are in hardware AWS cannot access. The bank owns the key material, AWS just provides the physical hardware and networking.

🚨 Exam Trap — If You Lose Your HSM Keys, They're Gone Forever: Unlike KMS (where AWS has backup mechanisms), if you lose the HSM credentials and have no backup, your encrypted data is permanently inaccessible. CloudHSM is single-tenant — AWS cannot recover your keys. Always deploy an HSM cluster (minimum 2 nodes) and maintain key backups in a separate HSM cluster.
🎯 SAA-C03 Exam Tips:
• "FIPS 140-2 Level 3, regulatory requirement for HSM" → AWS CloudHSM
• "AWS must not have access to encryption keys" → CloudHSM (KMS doesn't meet this)
• "TDE (Transparent Data Encryption) for Oracle/SQL Server on AWS" → CloudHSM
• "KMS-compatible but HSM-backed" → KMS Custom Key Store backed by CloudHSM
• CloudHSM is much more expensive than KMS — exam uses cost/regulatory tradeoff questions

Amazon Cognito
User Pools (AuthN) · Identity Pools (AuthZ) · Social Login · MFA · Millions of Users

Amazon Cognito lets you add user sign-up, sign-in, and access control to web/mobile apps in minutes. It handles the authentication heavy lifting — passwords, MFA, social logins (Google, Facebook, Apple), SAML federation — so you don't build it yourself. It scales to hundreds of millions of users.

💡 Two Components — You MUST Know the Difference:
User Pools = Authentication (AuthN). Who are you? Sign-up/sign-in, password management, MFA, JWT tokens. Think: user directory.
Identity Pools = Authorization (AuthZ). What can you access? Exchanges a User Pool JWT (or social token) for temporary AWS credentials (via STS). Grants access to AWS resources directly.

Architecture — Web App with Social Login + S3 Access

📱 Real-World Use Case
Fitness App — 5 Million Users, Google & Apple Login

A fitness app needs user registration, login with Google/Apple, MFA for premium users, and the ability to upload workout videos directly to their S3 bucket from the mobile app. With Cognito User Pool, they get sign-up/sign-in, Google/Apple OAuth federation, and MFA out of the box. With Cognito Identity Pool, authenticated users receive temporary AWS credentials scoped to their own S3 "folder" (prefix) — the mobile app uploads directly to S3 without going through a backend server. Zero auth code written from scratch. Scales to 5M users automatically.

🎯 SAA-C03 Exam Tips:
• "User sign-up / sign-in for web or mobile app" → Cognito User Pool
• "Grant AWS resource access (S3, DynamoDB) to app users" → Cognito Identity Pool
• "Social login (Google, Facebook, Apple)" → Cognito User Pool federated IdP
• "Mobile app directly access S3 — no backend server" → Identity Pool → STS temp creds
• "Millions of users" → Cognito is designed for this scale
Amazon Detective
Security Investigation · Root Cause Analysis · Graph Analysis · GuardDuty Integration

Amazon Detective uses machine learning and graph analysis to help you investigate security incidents and find the root cause quickly. It automatically collects log data from your AWS resources (VPC Flow Logs, CloudTrail, GuardDuty findings) and builds an interactive graph model — so when GuardDuty alerts on suspicious activity, Detective helps you understand who did what, when, and from where.

💡 The Key Distinction — Detective vs GuardDuty vs Security Hub:
GuardDuty = Detects threats (raises the alert — "This looks suspicious!")
Amazon Detective = Investigates the alert (answers "How did this happen? What's the blast radius?")
Security Hub = Aggregates findings from GuardDuty, Inspector, Macie etc. into a single dashboard

Architecture — Security Investigation Flow

🕵️ Real-World Use Case
Security Incident — Compromised IAM Credential Investigation

GuardDuty raises a UnauthorizedAccess:IAMUser/TorIPCaller finding. Someone is using a compromised IAM access key from a Tor exit node. The security team pivots to Amazon Detective with one click from the GuardDuty console. Detective's graph immediately shows: which IAM user, all API calls made in the last 30 days, which EC2 instances accessed, S3 buckets listed, and the originating IPs. What would take days of log analysis takes 10 minutes. The team revokes the key and patches the exfiltrated data scope — all before the attacker can extract anything meaningful.

🎯 SAA-C03 Exam Tips:
• "Investigate root cause of a security finding" → Amazon Detective
• "Understand the scope and impact of a security incident" → Amazon Detective
• "Detect threats in real time" → GuardDuty (not Detective)
• Detective consumes: VPC Flow Logs, CloudTrail, GuardDuty, EKS audit logs
• No agents to install — fully managed, ML-based, passive analysis

AWS Firewall Manager
Centralised WAF · Shield · Security Groups · Org-wide · Multi-Account

AWS Firewall Manager lets you centrally configure and manage firewall rules across all accounts and applications in your AWS Organization. Instead of configuring WAF rules in 50 different AWS accounts, you define the policy once in Firewall Manager and it enforces it everywhere — new accounts included, automatically.

💡 The Key Insight — Why Firewall Manager Exists:
Large enterprises have 100s of AWS accounts. Without Firewall Manager, security teams must manually configure WAF rules in every account — and new accounts start unprotected. Firewall Manager solves this: define once, enforce everywhere, auto-remediate violations.

Architecture — Organisation-Wide WAF Policy

🏢 Real-World Use Case
Global Enterprise — 200 AWS Accounts, WAF in Every One

A global retail company has 200 AWS accounts across business units. After a WAF bypass attack on one account, leadership mandates WAF rules on every public ALB and CloudFront distribution — across all 200 accounts. The security team uses AWS Firewall Manager to define a WAF policy with OWASP Top 10 rules. Firewall Manager automatically applies and enforces the policy across all 200 accounts. When a developer accidentally removes a WAF rule, Firewall Manager auto-remediates within minutes. New accounts automatically inherit protection. One policy, zero gaps.

🎯 SAA-C03 Exam Tips:
• "Centrally manage WAF rules across multiple AWS accounts" → AWS Firewall Manager
• "Automatically protect new accounts in an AWS Organisation" → Firewall Manager
• "Ensure all ALBs have WAF enabled — auto-remediate violations" → Firewall Manager
• Requires: AWS Organizations + Designated Firewall Manager admin account

Question 01 / 12 — Amazon Translate
A company wants to add real-time chat translation to their customer support platform. Support agents speak English only, but customers chat in 15 different languages. The solution must not require the company to manage any ML infrastructure. Which is the MOST appropriate solution?
A Train a custom neural network model in Amazon SageMaker and deploy it behind an API Gateway endpoint.
B Use Amazon Translate's real-time API to translate each incoming message before displaying it to the agent.
C Use Amazon Transcribe to transcribe messages and Amazon Comprehend to translate them.
D Store all messages in S3 and run an Amazon Translate batch job every 5 minutes.
✅ Answer: B
B is correct — Amazon Translate's synchronous (real-time) API is purpose-built for this. No ML infrastructure to manage. A is wrong — SageMaker requires ML expertise and infra management. C is wrong — Transcribe converts speech to text (not text-to-text), and Comprehend does NLP/sentiment analysis, not translation. D is wrong — batch jobs every 5 minutes aren't real-time (messages would be 5 minutes stale).
Question 02 / 12 — Elastic Transcoder
A media company stores raw video uploads in an Amazon S3 bucket. When a video is uploaded, they need it automatically converted into three formats: 1080p MP4 for web, 720p MP4 for tablet, and 360p MP4 for mobile. The engineering team should receive an email when each conversion job completes or fails. Which architecture meets these requirements with the LEAST operational overhead?
A S3 Event → Lambda → Elastic Transcoder Job (3 presets) → SNS notification on completion.
B Deploy FFmpeg on EC2 Auto Scaling Group, triggered by S3 events. EC2 sends SNS notification after conversion.
C Use Amazon Kinesis Video Streams to ingest uploads and apply transformations using Rekognition.
D S3 Event → AWS Glue job to transcode the video → SES email on completion.
✅ Answer: A
A is correct — S3 event notification triggers Lambda which creates an Elastic Transcoder job with 3 output presets. Elastic Transcoder natively supports SNS notifications on job complete/error. Zero servers. B has high operational overhead (managing EC2 fleet). C is wrong — KVS is for live video streaming, not file format conversion. D is wrong — AWS Glue is an ETL service for data (not video transcoding).
Question 03 / 12 — Kinesis Video Streams
A home security company manufactures smart doorbell cameras. Customers expect to see a live video stream in their mobile app and receive alerts when motion is detected. The backend must handle 2 million concurrent camera streams and perform real-time face recognition. Which combination of AWS services should the Solutions Architect recommend?
A Amazon Elastic Transcoder + Amazon Rekognition Image
B Amazon Kinesis Video Streams + Amazon Rekognition Video
C Amazon Kinesis Data Streams + AWS Lambda + OpenCV on EC2
D Amazon S3 video upload + Amazon Rekognition Video (batch)
✅ Answer: B
B is correct — Kinesis Video Streams ingests live streams from millions of cameras. Rekognition Video natively integrates with KVS for real-time face/object detection. KVS WebRTC can power the live view in the mobile app. A is wrong — Elastic Transcoder is for file conversion (not live streaming). C requires managing EC2 infrastructure for OpenCV — high operational overhead. D is batch processing — not real-time (5-minute delay minimum).
Question 04 / 12 — AWS Artifact
A healthcare startup's compliance officer needs to provide their auditors with evidence that AWS meets HIPAA requirements, and also needs to formally sign a Business Associate Agreement (BAA) with AWS before handling Protected Health Information (PHI). What is the SIMPLEST way to accomplish both tasks?
A Contact AWS Support and request the SOC 2 report and BAA via a support ticket.
B Use AWS Audit Manager to generate a HIPAA compliance report and sign the BAA there.
C Use AWS Artifact to download the HIPAA Eligibility documentation and sign the BAA agreement.
D Use AWS Config to assess HIPAA compliance and generate an agreement from the Config console.
✅ Answer: C
C is correct — AWS Artifact is specifically designed for this: download AWS compliance reports (HIPAA eligibility, SOC, PCI, ISO) and sign legal agreements (BAA, NDA). It's self-service, no support ticket needed. A is wrong — opening tickets is unnecessary; Artifact is self-service. B is wrong — Audit Manager collects evidence about YOUR environment's compliance, not AWS's compliance documents. D is wrong — AWS Config assesses resource configuration, not compliance document retrieval.
Question 05 / 12 — AWS Audit Manager
A financial services company undergoes a PCI DSS audit every quarter. Their compliance team spends 6 weeks before each audit manually pulling CloudTrail logs, security group configurations, and IAM reports to prove compliance. Which service would MOST reduce this manual effort?
A AWS Artifact — download the PCI DSS report from AWS
B AWS Audit Manager with the PCI DSS framework — continuously collect and organise evidence
C AWS Config with a custom rule set mapped to PCI DSS controls
D Amazon Inspector scans for PCI DSS vulnerabilities and generates reports
✅ Answer: B
B is correct — Audit Manager continuously collects evidence (CloudTrail, Config, Security Hub) and maps it to PCI DSS controls automatically. At audit time, you export a pre-organised evidence package. A is wrong — Artifact gives you AWS's PCI DSS report (about AWS), not evidence about your environment. C is a component that feeds INTO Audit Manager, but doesn't organise/package audit evidence. D — Inspector finds software vulnerabilities, not a comprehensive compliance evidence collector.
Question 06 / 12 — AWS Certificate Manager
A company is deploying a web application using Amazon CloudFront in the us-west-2 region. They want to use a custom domain name (app.example.com) with HTTPS. They request an ACM certificate in us-west-2 and try to attach it to CloudFront, but it is not appearing as an option. What is the MOST likely cause?
A CloudFront does not support custom SSL certificates.
B The ACM certificate must be requested in us-east-1 (N. Virginia) to be used with CloudFront.
C The domain must be registered in Route 53 to use ACM with CloudFront.
D ACM certificates cannot be used with CloudFront — they are only for ALBs.
✅ Answer: B
B is correct — This is one of the most commonly tested ACM facts: CloudFront is a global service, and ACM certificates for CloudFront must ALWAYS be requested in us-east-1 (N. Virginia), regardless of where your origin or users are. Certs in any other region simply won't appear when configuring CloudFront. A, C, D are all false. The domain does not need to be in Route 53 (any registrar works), and ACM absolutely supports CloudFront.
Question 07 / 12 — AWS CloudHSM
A government agency requires that all encryption keys for classified data must be stored in FIPS 140-2 Level 3 certified hardware that the agency exclusively controls. AWS must have zero ability to access or recover these keys. Which solution meets this requirement?
A AWS KMS with a Customer Managed Key (CMK)
B AWS KMS with an AWS Managed Key
C AWS CloudHSM with a dedicated HSM cluster
D AWS KMS Custom Key Store backed by CloudHSM, managed by AWS
✅ Answer: C
C is correct — CloudHSM provides single-tenant dedicated HSM hardware that is FIPS 140-2 Level 3 certified. Critically, AWS has ZERO access to key material — only the customer controls HSM users and keys. A is wrong — KMS CMKs are on AWS-managed hardware (multi-tenant in the HSM layer). AWS can theoretically access KMS key material (though they have policies against it). B is wrong — AWS Managed Keys are even more AWS-controlled. D describes a KMS Custom Key Store, but "managed by AWS" in the answer makes it wrong — you manage the HSM cluster yourself.
Question 08 / 12 — Amazon Cognito
A mobile gaming company wants users to sign in with their Google account, and then have the mobile app directly upload their game replay videos to an S3 bucket (the user's own folder only). There should be NO backend server involved in the upload process. Which Cognito feature enables direct S3 uploads from mobile?
A Cognito User Pool — returns a JWT token that the app uses to call S3 API directly
B Cognito Identity Pool — exchanges the Google token for temporary AWS credentials (IAM role) scoped to the user's S3 prefix
C Cognito User Pool with an S3 trigger Lambda that generates a pre-signed URL
D Cognito Sync — synchronises S3 data to mobile devices automatically
✅ Answer: B
B is correct — Cognito Identity Pool takes the Google OAuth token (or Cognito User Pool JWT) and exchanges it for temporary AWS credentials via STS AssumeRoleWithWebIdentity. The IAM role policy can scope access to only the user's S3 prefix (using cognito-identity.amazonaws.com:sub). The app uploads directly to S3 — zero backend. A is wrong — JWTs from User Pools are for authenticating to your app's APIs (API Gateway), not for AWS service APIs. C introduces a backend Lambda, violating the "no backend server" requirement. D — Cognito Sync is for syncing app data (key-value), not file uploads.
Question 09 / 12 — Amazon Detective
GuardDuty has raised a high-severity finding: an EC2 instance is communicating with a known command-and-control server. The security team needs to understand WHICH IAM role launched this instance, WHAT other API calls were made from this role in the last 30 days, and WHICH other resources may be affected. Which service is purpose-built for this investigation?
A AWS CloudTrail — manually search through event history for the IAM role's API calls
B Amazon GuardDuty — drill into the finding details for the full investigation
C Amazon Detective — use graph analysis to map the entity relationships and timeline
D AWS Security Hub — open the finding and view remediation recommendations
✅ Answer: C
C is correct — Amazon Detective is specifically built for investigation. It builds an ML-powered graph model from CloudTrail, VPC Flow Logs, and GuardDuty findings, letting you quickly see entity relationships, timelines, and blast radius. A is wrong — manually searching CloudTrail is extremely time-consuming and error-prone. B is wrong — GuardDuty raises the alert but doesn't provide deep investigative graph analysis. D is wrong — Security Hub aggregates findings but is a dashboard, not an investigation tool.
Question 10 / 12 — AWS Directory Service
A large enterprise is migrating to AWS. They have 10,000 employees already using Microsoft Active Directory on-premises. They want employees to use their existing Windows credentials to sign in to AWS services and access domain-joined EC2 Windows instances. The on-premises AD must remain the authoritative source — no user accounts should be replicated to AWS. Which Directory Service option is BEST?
A AWS Managed Microsoft AD — deploy a full managed AD in AWS and set up a two-way trust
B Simple AD — deploy Samba 4 compatible directory in AWS
C AD Connector — proxy authentication requests to the on-premises AD without replicating users
D Amazon Cognito — federate with the on-premises AD using SAML 2.0
✅ Answer: C
C is correct — AD Connector is a proxy: all authentication requests from AWS are forwarded to the on-premises AD. No user data is stored in AWS. On-prem AD remains the single source of truth. EC2 Windows instances can domain-join through it. A is wrong — Managed Microsoft AD replicates users to AWS (not desired), and sets up a trust (more complex than needed). B is wrong — Simple AD can't proxy to on-premises AD and doesn't support trusts. D is wrong — Cognito is for application-level user authentication, not corporate domain joining for EC2 or AWS Console SSO at scale.
Question 11 / 12 — AWS Firewall Manager
A company has 150 AWS accounts in their AWS Organization. The security team must ensure that all Application Load Balancers across all accounts have a specific set of AWS WAF rules applied. New accounts created in the future must automatically inherit these WAF rules. Non-compliant resources must be automatically remediated. Which service meets ALL these requirements?
A Deploy an AWS WAF WebACL in each account using AWS CloudFormation StackSets
B Use AWS Security Hub to detect missing WAF rules and trigger a Lambda to apply them
C Use AWS Firewall Manager with a WAF policy applied to the entire AWS Organization
D Use AWS Config with a custom rule to check WAF association and a remediation action
✅ Answer: C
C is correct — Firewall Manager is purpose-built for this exact scenario: centrally define WAF policies, apply them to all accounts in an AWS Organization, auto-enroll new accounts, and auto-remediate violations. A is wrong — CloudFormation StackSets deploys initially but doesn't continuously enforce or auto-remediate. B involves custom engineering with Security Hub + Lambda — high complexity. D is partial — Config can detect non-compliance but lacks the centralised org-wide WAF policy enforcement that Firewall Manager provides natively.
Question 12 / 12 — Amazon GuardDuty
A company's AWS account has been running for 2 years with no threat detection in place. Their CTO wants to enable intelligent threat detection that analyses CloudTrail, VPC Flow Logs, and DNS logs for malicious activity — with ZERO agent installation and ZERO log pipeline configuration. When a threat is found, on-call engineers must be notified via Slack within 5 minutes. What is the SIMPLEST architecture?
A Set up a SIEM with CloudWatch agent on all EC2 instances, export logs to Splunk, configure Splunk alerts to post to Slack.
B Enable Amazon GuardDuty → EventBridge rule on GuardDuty findings → SNS topic → Lambda → Slack webhook.
C Enable AWS Security Hub → subscribe to the GuardDuty integration → export findings to S3 → trigger Lambda every 5 minutes.
D Configure CloudTrail to deliver logs to S3 → EMR job to analyse logs for threats → SNS notification.
✅ Answer: B
B is correct — GuardDuty is one-click enable, no agents, no log configuration. It natively analyses CloudTrail, VPC Flow Logs, and DNS. EventBridge has built-in GuardDuty finding events, which trigger SNS → Lambda → Slack webhook. Near-real-time notification (under 5 minutes). A requires installing agents, managing Splunk, and complex infrastructure — contradicts "simplest." C introduces unnecessary S3 polling (5-minute lag) and Security Hub is an aggregator, not the threat detector. D requires building a custom analytics pipeline — extremely complex compared to just enabling GuardDuty.
