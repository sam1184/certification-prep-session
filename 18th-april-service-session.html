AWS Security Hub
Central Security Findings · CSPM · Multi-Account Aggregation · Compliance Standards

Security Hub is your single pane of glass for AWS security. It aggregates, normalises, and prioritises security findings from dozens of AWS services (GuardDuty, Inspector, Macie, Config, IAM Access Analyzer) and third-party tools into one dashboard. Think of it as the CEO dashboard for your entire security team — instead of logging into 6 different services, everything lands in Security Hub with a unified severity score.

Architecture — Multi-Account Security Aggregation

FinTech — Centralised Security Operations Centre (SOC)

A bank runs 40 AWS accounts across dev, staging, and production. Instead of having the security team log into each account individually, they enable Security Hub in a dedicated Security account as the aggregator. All 40 member accounts forward findings. The SOC sees one dashboard with a compliance score against PCI DSS. When a critical finding arrives (exposed S3 bucket), EventBridge routes it to a Lambda that automatically enables S3 Block Public Access and posts to the #security-alerts Slack channel. Mean time to remediation drops from hours to seconds.

🎯 SAA-C03 Exam Tips:

AWS Shield
DDoS Protection · Standard (Free) vs Advanced ($3K/mo) · Always-on Detection · SRT Team

AWS Shield protects your AWS resources from Distributed Denial of Service (DDoS) attacks — where attackers flood your application with millions of fake requests to take it offline. Shield comes in two tiers: Standard (free, automatically protects all AWS resources) and Advanced (paid, for high-value applications needing 24/7 expert DDoS response).

💡 Mental Model — The Two Shields:
Shield Standard = The seatbelt in your car. It's always there. You don't pay extra. It handles 96% of all volumetric and protocol DDoS attacks automatically (SYN floods, UDP floods, reflection attacks).

Shield Advanced = Hiring a personal bodyguard team. $3,000/month minimum. Adds application-layer (L7) DDoS protection, real-time attack visibility, DDoS cost protection (AWS credits your bill if an attack causes scaling), and 24/7 access to the AWS Shield Response Team (SRT).
Free
Shield Standard
Automatic L3/L4 DDoS protection for all AWS customers. Always on, zero config needed.
$3K/mo
Shield Advanced
L3/L4/L7 protection + SRT + DDoS cost protection + near real-time visibility + WAF integration

Architecture — DDoS Protection Layers

🎮 Real-World Use Case
Online Gaming Platform — Staying Online During Launch Day

A gaming company launches a multiplayer game. On day 1, a competitor orchestrates a DDoS attack sending 50 Gbps of UDP traffic to their servers. Shield Standard (included free) automatically detects and absorbs the volumetric attack at the AWS network edge — players experience no downtime. The company then upgrades to Shield Advanced before their next major launch — if attackers switch to L7 HTTP flooding, WAF rules kick in, and the SRT team calls them proactively. Any EC2 scaling triggered by attack traffic is credited back by AWS's DDoS cost protection.

🚨 Exam Trap — Shield vs WAF: Shield protects against network/transport layer DDoS (volumetric attacks). WAF protects against application layer attacks (SQL injection, XSS). The exam may describe both as "protection." Shield Advanced + WAF together = comprehensive protection. Shield alone cannot block SQL injection. WAF alone cannot block a 1Tbps UDP flood.
🎯 SAA-C03 Exam Tips:
• "Protect against DDoS automatically at no cost" → Shield Standard (always on, free)
• "24/7 DDoS response team + cost protection" → Shield Advanced
• Shield Advanced protects: CloudFront, ALB, NLB, Route 53, Elastic IP, Global Accelerator
• "DDoS cost protection" is a key Shield Advanced differentiator — exam loves this
• Shield Standard is automatic — you never configure or pay for it
AWS WAF
Web Application Firewall · L7 Rules · Managed Rules · Rate Limiting · Bot Control

AWS WAF (Web Application Firewall) filters HTTP/HTTPS traffic at Layer 7. Where Shield stops floods, WAF stops intelligent attacks: SQL injection, cross-site scripting (XSS), bad bots, and malicious IPs. You write rules (or use AWS-managed rule groups) that inspect each request — if a rule matches, you can Allow, Block, or Count it.

💡 Key Concepts — Web ACLs, Rules, Rule Groups:
Web ACL = The policy document — attached to CloudFront, ALB, API Gateway, AppSync, or Cognito User Pool.
Rule = A condition: "Block requests where query string contains 'DROP TABLE'"
Rule Group = A reusable collection of rules. AWS provides Managed Rule Groups (AWSManagedRulesCommonRuleSet, AWSManagedRulesSQLiRuleSet) — pre-built, auto-updated by AWS. No rule-writing needed.
SQLi
SQL Injection
Blocks attacks trying to manipulate your database queries
XSS
Cross-Site Script
Blocks scripts injected into pages to steal user sessions
Rate
Rate Limiting
Block IPs sending more than N requests per 5 minutes — stops brute force
Geo
Geo-Block
Block or allow traffic from specific countries via geo-match conditions

Architecture — WAF on CloudFront

🛒 Real-World Use Case
E-Commerce — Blocking Credential Stuffing & Bad Bots

A retail website's login endpoint is being targeted by bots running credential stuffing attacks — trying 10,000 username/password combinations stolen from other breaches. The team attaches a WAF Web ACL to their ALB with three rules: (1) AWS Managed SQLi Rule Set, (2) Rate Limit: block any IP making more than 100 requests/5 min to /login, (3) Bot Control Managed Rule Group to identify and block automated bots. Attackers are blocked at Layer 7 before they ever hit the application servers. WAF logs feed into CloudWatch for anomaly dashboards. Cost: ~$5/month per Web ACL + $0.60/million requests inspected.

🎯 SAA-C03 Exam Tips:
• "Block SQL injection / XSS on your web application" → AWS WAF
• WAF attaches to: CloudFront, ALB, API Gateway, AppSync, Cognito User Pool
• "Block requests from a specific country" → WAF Geo-match rule
• "Rate limit API calls per IP" → WAF Rate-based rule
• WAF is deployed at us-east-1 for CloudFront (global), or regional for ALB/API Gateway
• WAF alone doesn't stop DDoS floods — pair with Shield Advanced for full protection

Amazon API Gateway
Managed API · REST / HTTP / WebSocket · Auth · Throttling · Caching · Lambda Integration

API Gateway is the front door for your backend services. It receives API requests from clients, authenticates them, throttles them, optionally caches responses, and routes them to backend targets: Lambda functions, EC2, ECS, any HTTP endpoint. Think of it as the intelligent doorman for your APIs — so your backend never gets overwhelmed and never gets accessed without authorisation.

💡 Three API Types — Know the Differences:
REST API = Full-featured: IAM auth, Cognito, Lambda Authorizer, API Keys, Usage Plans, caching, request/response transformation. Use for complex APIs.
HTTP API = Lighter, faster, cheaper (~70% lower cost). Supports JWT auth, Lambda, and proxy only. No built-in caching. Use for simple Lambda/HTTP backends.
WebSocket API = Persistent two-way connections. Routes messages to different Lambda functions. Use for real-time apps: chat, live dashboards, games.

Architecture — Serverless API with Lambda

📱 Real-World Use Case
Mobile Banking App — Serverless Transactions API

A fintech app has millions of users checking balances and making transfers. They build a serverless API: mobile app → API Gateway (validates Cognito JWT token, throttles to 5 req/sec per user, caches GET /balance for 60 seconds) → Lambda (executes transaction logic) → DynamoDB. API Gateway handles 500,000 requests/day at $1.75/day. The balance cache alone reduces Lambda invocations by 80%. API versioning (v1/ → v2/) allows seamless migration without breaking old app versions. No load balancer, no server fleet, no patching.

🚨 Exam Trap — REST API vs HTTP API: The exam may ask "lowest cost API Gateway option for a Lambda proxy." Answer = HTTP API (70% cheaper, but no caching, no request transformation). If the question mentions "caching," "API Keys + Usage Plans," or "request body mapping," it must be REST API. WebSocket is only for persistent bidirectional connections (chat, real-time dashboards).
🎯 SAA-C03 Exam Tips:
• "Throttle API calls to prevent Lambda/backend overload" → API Gateway throttling
• "Cache API responses to reduce backend calls" → API Gateway caching (REST only)
• "Real-time bidirectional connection (chat/gaming)" → WebSocket API
• "Cheapest managed API" → HTTP API
• API Gateway + Lambda = most common serverless pattern on the exam

Amazon Cognito

Amazon Cognito manages identity and access for web and mobile applications. It handles user sign-up, sign-in, and authorisation at millions-of-users scale. There are two distinct concepts: User Pools (who are you? — authentication) and Identity Pools (what can you do with AWS? — authorisation via temporary credentials).

💡 User Pool vs Identity Pool — Exam Core Concept:

Think: "Give my mobile app users direct-but-scoped AWS access without a backend server."

Amazon Detective is specifically designed for security investigation — it correlates GuardDuty findings, CloudTrail, and VPC Flow Logs into interactive graphs with timelines and baseline comparisons. Security Hub aggregates findings but doesn't provide deep investigation. GuardDuty detects threats but doesn't provide deep graph-based investigation. CloudTrail Insights detects anomalies in API activity but doesn't provide the correlation graph that Detective does.

A company wants to apply WAF rules, Shield Advanced protection, and security group policies consistently across all 80 accounts in their AWS Organisation. New accounts should automatically receive these policies when they join. Which service should they use?
A Configure WAF and Shield Advanced in each account individually
B AWS Firewall Manager from the management account
C AWS Security Hub with automated remediation
D AWS Config with organisation-wide conformance packs

AWS Firewall Manager is designed exactly for this scenario — it deploys WAF, Shield Advanced, and security group policies across all accounts in an AWS Organization, and automatically applies policies to new accounts when they join. Configuring each account individually doesn't scale. Security Hub aggregates findings but doesn't deploy firewall rules. Config conformance packs detect non-compliance but don't deploy WAF rules.

Question 12 of 15
A company wants to add HTTPS to their CloudFront distribution for a domain hosted in ap-southeast-1. Where must the ACM certificate be provisioned?
A ap-southeast-1 (same region as the origin)
B us-east-1 (N. Virginia) — required for all CloudFront certs
C Any region — ACM certificates are global
D us-west-2 (CloudFront's home region)
Correct Answer: B
ACM certificates for CloudFront MUST always be in us-east-1 (N. Virginia) — this is a hard AWS requirement regardless of where your origin or users are located. This is one of the most commonly tested ACM/CloudFront facts. For ALB, the cert must be in the same region as the ALB. CloudFront is a global service but reads its certificates from us-east-1 only.
Question 13 of 15
A company's employees need to use their existing on-premises Active Directory credentials to access AWS WorkSpaces. The IT team wants no user data replicated to AWS — all authentication should proxy to the existing AD. Which Directory Service option is correct?
A AWS Managed Microsoft AD with forest trust
B Simple AD
C AD Connector
D Amazon Cognito SAML federation
Correct Answer: C

Simple AD is a basic Samba directory with no connection to Microsoft AD. Cognito SAML is for web/mobile apps, not WorkSpaces desktop authentication.
Question 14 of 15
Amazon GuardDuty raises an alert that an EC2 instance has been communicating with a known Tor exit node. The security team wants to automatically isolate the instance and revoke its IAM role credentials within seconds of the finding. What is the correct architecture?
A GuardDuty → CloudWatch Logs → CloudWatch Alarm → SNS → Manual intervention
B GuardDuty → EventBridge → Lambda (isolate SG + revoke role credentials)
C GuardDuty → Security Hub → Config → Auto-remediation
D GuardDuty → Amazon Detective → Lambda automated response
Correct Answer: B
GuardDuty → EventBridge → Lambda is the standard automated incident response pattern. GuardDuty publishes findings as EventBridge events. An EventBridge rule matches the finding type/severity and triggers a Lambda function that modifies the EC2 security group (isolate) and revokes IAM role credentials. This is the canonical "automated remediation" architecture on the SAA-C03 exam.

A retail company wants to predict demand for 10,000 products over the next 90 days to optimise inventory ordering. They have 5 years of daily sales data in S3 and no machine learning team. Which service should they use?
A Amazon SageMaker Autopilot with tabular data
B Amazon Forecast
C Amazon Comprehend with time-series analysis
D Amazon QuickSight ML Insights
Correct Answer: B
Amazon Forecast is the purpose-built managed service for time-series forecasting — exactly this use case. It automatically trains and evaluates multiple ML models (DeepAR+, ARIMA, ETS) and returns demand predictions with confidence intervals. No ML expertise required — just upload historical data and related information (promotions, holidays). SageMaker Autopilot works for tabular ML tasks but requires ML knowledge for time-series tuning. Comprehend is NLP. QuickSight ML Insights provides anomaly detection, not demand forecasting.
